# 金灯塔胜算 Skill DEMO

## 描述

本 skill 集成飞书授权体系与胜算业务系统，支持用户通过自然语言查询 BI 数据报表。
Claude 自动识别用户意图，调用对应系统 API 获取数据，生成标准化 HTML 可视化报表，
上传至 OSS 并返回预览链接及数据总结建议，最终可将报表发布至胜算平台或通过定时任务自动推送。

---

## 触发条件

以下场景 Claude 应主动激活本 skill：

- 用户提及"报表"、"数据看板"、"BI"、"统计"、"分析"等关键词
- 用户询问某系统的业务数据（如销售额、订单量、用户数等）
- 用户要求"生成图表"、"导出数据"、"发布报表"、"定时推送"

## 初始化流程（首次使用 / 会话开始时执行一次）

### Step 1：飞书授权
- 向用户展示所需权限列表，提供一键全部确认入口
- 所需权限：
  - 获取用户手机号
  - 发送飞书消息
  - 创建/管理定时任务
- 用户逐项或一键确认后继续；任意必要权限拒绝则阻塞后续流程，明确告知缺少哪项权限

### Step 2：获取用户身份
- 调用飞书 API 获取当前用户绑定的手机号，作为胜算系统全程用户标识
- 失败则提示用户确认飞书账号手机号绑定状态，不进入后续流程

### Step 3：加载系统配置
- 调用胜算 API：`GET /auth/supported-systems`
- 响应包含：
  - `systems[]`：支持的系统名称列表
  - `oss_domain`：OSS 访问域名
  - `oss_api`：OSS 文件上传接口地址
  - `oss_static_domain`：静态资源（ECharts 等 JS 库）的 OSS 托管域名（写死引用路径用）
- Claude 将以上配置全部存入会话上下文，整个会话复用，不重复请求

### Step 4：系统授权选择
- Claude 向用户展示 `systems[]` 列表，提示选择目标系统
- 用户指定后，调用：`POST /auth/system-token`
  - 入参：`{ phone, system_name }`
  - 成功：返回 `{ token, expires_at }`，存入上下文
  - 失败（无权限）：明确告知用户缺少该系统权限，建议联系管理员申请，终止流程
- **Token 过期处理**：每次调用系统 API 前检查 `expires_at`，若已过期或距过期不足 5 分钟则自动静默刷新，对用户透明

### Step 5：加载系统 API 元数据
- 使用 token 调用：`GET /system/api-registry`
- 响应包含每个接口的完整文档信息：
  - `api_id`：接口唯一标识
  - `name`：接口用途说明
  - `method`：请求方式（GET / POST）
  - `endpoint`：接口完整地址
  - `params[]`：参数列表，每项含：字段名、类型、是否必填、取值说明、示例值
  - `response_schema`：响应数据结构说明（字段名、类型、含义）
- Claude 完整解析并存储 API 注册表，后续意图匹配、参数提取、数据解析均依赖此文档，确保调用无歧义

### Step 6：初始化完成
- Claude 向用户确认已就绪，说明当前选择的系统名称
- 提示用户可以开始描述报表需求，或设置定时任务

## 核心流程：BI 报表生成

### Step 1：意图识别与接口匹配
- 用户以自然语言描述需求（如"给我看上个月各渠道的销售额趋势"）
- Claude 结合 API 注册表中的 `name` 和 `response_schema`，匹配最相关的接口
- 匹配不明确时，Claude 列出候选接口供用户确认，不猜测、不静默选择

### Step 2：参数提取与补全
- 从用户输入中提取接口所需参数（时间范围、维度、过滤条件等）
- 必填参数缺失时，逐一向用户追问，不得用默认值静默填充
- 向用户展示最终将发起的查询条件，确认后再调用接口

### Step 3：调用系统 API 获取数据
- 按接口文档要求，携带 token 发起请求
- 返回数据为空：告知用户并询问是否调整查询条件，不继续生成报表
- 返回错误：展示错误码与描述，不继续后续步骤

### Step 4：生成 HTML 报表文件

> HTML 文件为完整独立可运行文件，结构与规范如下：

#### 页面结构（从上至下）
```
┌─────────────────────────────────────┐
│           报表标题区                  │
│  {系统名称} · {报表名称} · {数据时间范围} │
├─────────────────────────────────────┤
│           筛选条件区                  │
│  时间范围 / 维度 / 其他条件            │
│  [查询] 按钮 → 触发真实 API 接口重新查询 │
├─────────────────────────────────────┤
│           BI 内容区                  │
│  上方：ECharts 图表（折线/柱状/饼图等）  │
│  下方：对应数据明细表格                │
├─────────────────────────────────────┤
│           底部总结区                  │
│  数据总结（关键指标、趋势描述）          │
│  建议（基于数据的可执行建议）            │
│  生成时间 · 数据来源                   │
└─────────────────────────────────────┘
```

#### 技术规范

**JS 库引用**
- ECharts 及其他所有需要 script 引入的 JS 库，统一使用 OSS 托管地址（`oss_static_domain`）
- 格式示例（地址后续补充，当前预留占位）：
```html
  <!-- ECharts -->
  <script src="{oss_static_domain}/libs/echarts/echarts.min.js"></script>
  <!-- 其他库按需同规则引入 -->
```
- **禁止使用任何 CDN 外链（jsdelivr / unpkg / cdnjs 等）**

**响应式兼容**
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- PC 端：筛选区横向排列，图表与表格上下排列，最大宽度 1200px 居中
- 移动端（≤768px）：筛选区纵向堆叠，图表宽度 100%，表格横向滚动

**筛选区交互**
- 筛选条件变更后点击「查询」按钮，通过 `fetch` 携带当前筛选参数调用真实 API 接口
- 请求时显示加载状态（loading），防止重复点击
- 请求成功后更新图表与表格数据；失败则展示错误提示，不破坏已有数据

**图表类型选择规则**
- 时间趋势数据 → 折线图
- 分类对比数据 → 柱状图
- 占比构成数据 → 饼图 / 环形图
- 多维对比数据 → 分组柱状图
- 数据类型不明确时，Claude 向用户确认，不自动猜测

**样式要求**
- 整体风格：简洁商务，白底，主色调蓝灰系
- 字体：系统默认无衬线字体
- 表格：带斑马纹、表头固定、超出行数可滚动
- 图表：含图例、数据标签、tooltip hover 交互

### Step 5：上传 OSS

- 调用上下文中存储的 `oss_api`，上传 HTML 文件
- 文件命名规则：`{system_name}_{report_name}_{YYYYMMDD_HHmmss}.html`（防止覆盖）
- 请求方式与 Body 格式按 `oss_api` 接口文档执行
- 成功：返回 `{ preview_url }`，存入上下文 `last_report_url`
- 失败：明确提示上传失败原因，不输出任何链接，提供重试选项

### Step 6：输出报表结果

每次生成报表（含定时任务推送），统一按以下格式输出：
```
📊 **{报表名称}**

🔗 预览地址：{oss_preview_url}

📝 总结与建议：
{基于本次数据的关键指标描述、趋势分析、可执行建议}

⏱ 生成时间：{YYYY-MM-DD HH:mm:ss}
```

- 总结与建议由 Claude 基于实际返回数据生成，不得输出通用模板文字
- 询问用户是否需要发布到胜算，或设置定时推送

## 定时任务流程

### 触发条件
- 用户主动要求"定时发送"、"每天/每周推送"此报表

### Step 1：配置确认
- Claude 向用户确认：
  - 执行频率（每日 / 每周 / 自定义 Cron）
  - 执行时间
  - 推送目标（当前用户 / 指定飞书用户或群）
  - 使用的查询参数（时间范围是否动态偏移，如"始终查最近7天"）

### Step 2：注册定时任务
- 调用飞书定时任务 API 创建任务，携带上述配置
- 任务触发时，自动执行：Step 3（调用 API）→ Step 4（生成 HTML）→ Step 5（上传 OSS）→ 按统一格式推送飞书消息

### Step 3：确认反馈
- 返回任务 ID 与下次执行时间，提示用户可随时取消

## 发布流程

> 将已生成的报表持久化发布至胜算平台

### 触发条件
- 用户确认发布，或明确要求"发布"、"保存到胜算"

### Step 1：发布信息确认
- Claude 自动填充以下字段，用户可修改后确认：
  - 报表标题（默认：报表名称 + 数据时间范围）
  - 所属系统（默认：当前系统）

### Step 2：调用发布接口
- 调用胜算 API：`POST /report/publish`
  - 入参：`{ token, title, oss_url, system_name, created_by: phone }`
  - 成功：返回胜算平台报表详情页 URL
  - 失败：展示具体错误原因，提示重试

### Step 3：完成反馈
- 返回胜算平台报表链接
- 可选：通过飞书消息将链接推送给当前用户（若已授权）

## 错误处理规范

| 场景 | 处理方式 |
|------|----------|
| 飞书授权拒绝 | 明确告知缺少哪项权限，阻塞流程，引导手动开启 |
| 手机号获取失败 | 提示确认飞书账号绑定状态 |
| 无系统权限 | 明确告知缺少哪个系统权限，建议联系管理员 |
| Token 过期 | 自动静默刷新，对用户透明 |
| API 参数缺失 | 逐一追问用户，不静默填充默认值 |
| API 返回数据为空 | 告知用户，询问是否调整筛选条件 |
| API 调用失败 | 展示错误码与描述，不继续生成报表 |
| OSS 上传失败 | 提示失败原因，提供重试，不输出无效链接 |
| 定时任务执行失败 | 通过飞书消息通知用户，附错误说明 |
| 发布接口失败 | 展示失败原因，不影响已生成的 OSS 预览链接 |

## 会话上下文变量

| 变量 | 来源 | 用途 |
|------|------|------|
| `user_phone` | 飞书 API | 用户身份标识 |
| `oss_domain` | 胜算初始化接口 | OSS 文件访问 |
| `oss_api` | 胜算初始化接口 | 文件上传 |
| `oss_static_domain` | 胜算初始化接口 | JS 库静态资源引用地址 |
| `system_name` | 用户选择 | 当前操作系统标识 |
| `system_token` | 系统授权接口 | 调用系统 API 鉴权 |
| `token_expires_at` | 系统授权接口 | Token 过期自动刷新判断 |
| `api_registry` | 系统 API 注册表 | 意图匹配、参数提取、数据解析 |
| `last_report_url` | OSS 上传结果 | 发布流程复用，避免重复上传 |
